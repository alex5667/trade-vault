## 3. Причина: почему калибровка не дает результата

### Анализ dataset
```
Total rows: 357
OK=1: 0 (0.0%)
OK=0: 357 (100%)
r_mult != 0: 0 (все сделки имеют r_mult=0)
Score range: 0.120 - 0.270
Score mean: 0.268
```

### Проблема
Калибровка требует:
- `pass_rate >= 0.15` (минимум 15% сделок должны пройти gate)
- `tail_pass <= 0.18` (максимум 18% tail loss среди прошедших)
- Максимизация `meanR_pass`

**Текущая ситуация:**
- Все сделки имеют `ok=0` (не прошли gate)
- Все сделки имеют `r_mult=0` (нет закрытых сделок с результатами)
- **Вывод:** Либо сделки еще не закрыты, либо данные неполные

### Код калибровки
```python
# tools/calibrate_gate_params.py:128-131
if m["pass_rate"] < args.pass_min:  # pass_min=0.15 по умолчанию
    continue
if m["tail_pass"] > args.tail_max:  # tail_max=0.18 по умолчанию
    continue
```

**Результат:** Ни одна комбинация параметров не проходит ограничения, поэтому `best=null`.

## 4. Почему старые директории были пустыми

### Возможные причины:
1. **Redis stream был пуст** в момент запуска по расписанию (02:20, 03:30, 04:10 UTC)
2. **Экспорт не смог получить данные** из-за проблем с подключением
3. **State file указывал на несуществующие записи** (после очистки stream)
4. **Недостаточно закрытых сделок** для построения dataset

### Проверка логов
- Логи контейнеров не показывают детали запуска по расписанию
- Возможно, контейнеры не запускались или не логировали детали

## 5. Рекомендации и решения

### A. Для калибровки (of-nightly-calibrate)

#### Проблема: Нет закрытых сделок с результатами
**Решение 1:** Увеличить временное окно для закрытых сделок
```bash
# В docker-compose-timers.yml
TRADES_SINCE_HOURS=168  # вместо 72 (7 дней вместо 3)
```

**Решение 2:** Ослабить ограничения калибровки
```python
# В nightly_gate_calibrate_bundle.py, добавить параметры:
subprocess.run([
    sys.executable, "-m", "tools.calibrate_gate_params",
    "--dataset", dataset_out,
    "--out", calib_out,
    "--tail-max", "0.25",  # увеличить с 0.18
    "--pass-min", "0.05",  # уменьшить с 0.15
], check=True, capture_output=True, text=True)
```

**Решение 3:** Проверить, почему все сделки имеют `ok=0`
- Возможно, gate слишком строгий
- Проверить логику в `build_of_dataset.py`

### B. Для предотвращения пустых директорий

#### Решение 1: Добавить проверку данных перед запуском
```python
# В nightly_gate_calibrate_bundle.py, перед экспортом:
r = redis.Redis.from_url(redis_url, decode_responses=True)
stream_len = r.xlen(inputs_stream)
if stream_len < 100:
    status_data["status"] = "SKIPPED"
    status_data["reason"] = f"insufficient_data stream_len={stream_len}"
    raise SystemExit(f"insufficient_data stream_len={stream_len}")
```

#### Решение 2: Улучшить логирование
```python
# Добавить детальное логирование в каждый шаг:
import logging
logger = logging.getLogger(__name__)
logger.info(f"Step 1: Exporting inputs from stream {inputs_stream}...")
logger.info(f"Step 1: Exported {n} records")
```

#### Решение 3: Проверить state file перед экспортом
```python
# Проверить, что state file указывает на существующие записи:
if resume and state_file:
    last_id = _read_last_id(state_file)
    # Проверить, что last_id существует в stream
    if not r.exists(f"{inputs_stream}:{last_id}"):
        # Сбросить state file или использовать начало stream
        last_id = None
```
