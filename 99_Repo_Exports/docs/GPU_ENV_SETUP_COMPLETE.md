# ✅ Установка переменных окружения для GPU оптимизации

## 📊 Статус: Переменные окружения установлены

**Дата**: 2025-01-XX  
**Изменения**: Добавлены настройки для оптимизации использования GPU

---

## ✅ Установленные переменные

В файле `docker-compose.yml` в сервисе `multi-symbol-orderflow` добавлены:

```yaml
# ═══ GPU ОПТИМИЗАЦИЯ: Настройки батч-обработки для более частого использования GPU ═══
- CANDLE_BATCH_SIZE=5 # Размер батча свечей для GPU обработки (уменьшено для более частой обработки)
- CANDLE_BATCH_INTERVAL_SEC=2.0 # Интервал принудительной обработки батча в секундах
- GPU_ENABLED=true # Включить GPU ускорение
```

### Значения:
- **CANDLE_BATCH_SIZE**: `5` (вместо 10 по умолчанию)
- **CANDLE_BATCH_INTERVAL_SEC**: `2.0` (вместо 5.0 по умолчанию)
- **GPU_ENABLED**: `true`

---

### 1. Перезапуск сервиса для применения новых настроек:

```bash

docker compose down && docker compose up -d
```

### 2. Проверка применения настроек:

```bash

# Ожидаемый вывод:
```

### 3. Проверка логов для подтверждения:

```bash

# Должны быть логи:
# - "🚀 GPU acceleration enabled: ..."
# - Информация о размере батча и интервале обработки
```

## 📝 Использование переменных в коде

Переменные используются в:
- `python-worker/of/candle_of_worker.py`:
  - `self.batch_size = int(os.getenv('CANDLE_BATCH_SIZE', '10'))`
  - `self.batch_process_interval = float(os.getenv('CANDLE_BATCH_INTERVAL_SEC', '5.0'))`

---

## 🚨 Важные замечания

1. **Минимальные значения**:
   - `CANDLE_BATCH_SIZE`: рекомендуется минимум 3-5 (меньше = больше overhead)
   - `CANDLE_BATCH_INTERVAL_SEC`: рекомендуется минимум 1.0 секунда

2. **Баланс производительности**:
   - Слишком малые значения могут увеличить overhead
   - Слишком большие значения снижают использование GPU

3. **Мониторинг**:
   - Следите за использованием GPU через `nvidia-smi`
   - Проверяйте логи на наличие ошибок
