# Обновление отображения размера позиции в сигналах

## Что изменилось

Теперь в сигналах для криптовалюты отображается **дополнительная информация** о размере позиции:

### Старый формат:
```
🚨 🔴 BTCUSDT SHORT @ 93270.00, Volume 0.96 USDT
```

### Новый формат:
```
🚨 🔴 BTCUSDT SHORT @ 93270.00, Volume 0.96 USDT (0.96% депозита), с плечом 100x = 96.00 USDT
```

## Что показывает новый формат

1. **Volume 0.96 USDT** - размер позиции в долларах (маржа, которая используется)
2. **(0.96% депозита)** - какой процент от вашего депозита используется
3. **с плечом 100x = 96.00 USDT** - фактический размер контракта с учётом плеча

## Пример расчёта

При депозите **100 USDT** и плече **100x**:
- Volume **0.96 USDT** = маржа, которая резервируется
- Это **0.96%** от депозита (0.96 / 100 * 100%)
- С плечом 100x фактический размер контракта = **96 USDT** (0.96 * 100)

## Технические изменения

### Модифицированные файлы:

1. **`python-worker/services/pnl_math.py`**
   - Функция `calculate_position_size()` теперь возвращает `(lot, position_size_usd, deposit, leverage)` вместо `(lot, position_size_usd)`

2. **`python-worker/handlers/base_orderflow_handler.py`**
   - Метод `_publish_signal()` теперь сохраняет `deposit` и `leverage` в объекте signal

3. **`python-worker/core/unified_signal_formatter.py`**
   - Добавлены поля `deposit` и `leverage` в класс `Signal`
   - Метод `format_telegram_message()` использует эти значения из signal или из ENV

4. **`python-worker/core/crypto_signal_formatter.py`**
   - Логика для отображения процента депозита и размера с плечом уже была реализована ранее
   - Теперь получает актуальные значения deposit и leverage из signal

## Конфигурация

Значения берутся из переменных окружения:
```bash
ACCOUNT_DEPOSIT_USD=100      # Размер депозита в USDT
ACCOUNT_LEVERAGE=100         # Плечо (1:100)
RISK_PERCENT=5.0            # Процент риска на сделку
```

## Обратная совместимость

Изменения **полностью обратно совместимы**:
- Если в signal не переданы deposit/leverage, используются значения из ENV
- Старые обработчики сигналов продолжают работать без изменений
- Для инструментов типа XAUUSD/Forex ничего не изменилось
