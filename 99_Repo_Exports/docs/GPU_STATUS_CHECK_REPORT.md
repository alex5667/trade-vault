## 🔍 4. Анализ использования батч-обработки

### Важное наблюдение:

1. **`candle_of_worker.py` не используется напрямую**:
   - В `main_multi_symbol.py` используются handlers через `OrderFlowHandlerFactory`
   - Handlers используют `base_orderflow_handler.py`, который обрабатывает тики, а не свечи

2. **Батч-обработка в `candle_of_worker.py`**:
   - Класс `CandleOrderFlowWorker` имеет батч-обработку
   - НО используется только в `signal_processor.py` (legacy)
   - НЕ используется в текущем `main_multi_symbol.py`

3. **Текущие handlers**:
   - Используют `base_orderflow_handler.py` и его подклассы
   - Обрабатывают тики в реальном времени
   - GPU используется для вычислений (z-scores, robust z-score, etc.)

## 🎯 5. Где реально используется GPU

### Текущие места использования GPU:

1. ✅ **Robust Z-Score** (`base_orderflow_handler.py`):

2. ✅ **Rolling Window** (`featurizer.py`):

3. ✅ **Z-Scores** (`metrics/features.py`):

5. ⚠️ **Батч-обработка свечей** (`candle_of_worker.py`):
   - НЕ используется в текущем архитектуре (legacy код)
