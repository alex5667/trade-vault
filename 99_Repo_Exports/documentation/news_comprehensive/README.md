# Комплексная Документация по Сбору, Анализу и Использованию Новостей

## Обзор Системы

Новостной пайплайн представляет собой высокопроизводительную распределенную систему реального времени для сбора, интеллектуального анализа и интеграции новостной информации в процесс принятия торговых решений. Система разработана для обработки тысяч новостей в минуту с минимальной задержкой и максимальной надежностью.

## Компоненты Системы

### 1. Источники Новостей (News Sources)
- **RSS Feeds**: Структурированные новостные фиды от авторитетных источников
- **CryptoPanic API**: Специализированные крипто-новости с категоризацией
- **Financial Modeling Prep**: Финансовые новости и экономический календарь
- **NewsAPI**: Широкий охват новостных источников

### 2. Ингестор (News Ingestor)
- **Go News Ingestor**: Основной высокопроизводительный ингестор
- **Python Standby**: Резервный ингестор для отказоустойчивости
- **Дедупликация**: Предотвращение повторной обработки новостей
- **Нормализация**: Стандартизация форматов данных

### 3. Анализатор (News Analyzer)
- **LLM Анализ**: Google Gemini для оценки важности и категоризации
- **Оценка Риска**: 0-1 шкала влияния новости на рынок
- **Анализ Тегов**: Автоматическая категоризация по типам событий
- **Confidence Score**: Уверенность модели в анализе

### 4. Хранилище Фич (Feature Store)
- **Redis Streams**: Потоковая обработка с сохранением истории
- **EMA Агрегация**: Экспоненциальное сглаживание для стабильности
- **Многосимвольная Поддержка**: Агрегация данных по символам
- **TTL Управление**: Автоматическая очистка устаревших данных

### 5. Интеграция в Сигналы (Signal Integration)
- **Enricher Pattern**: Обогащение контекста сигналов новостными данными
- **Фильтрация**: Отсеивание сигналов во время высокой волатильности
- **Взвешивание**: Динамическая корректировка весов сигналов
- **Временные Ограничения**: Управление горизонтом торговли

## Ключевые Возможности

### Производительность
- **Задержка**: < 50ms для ingestion, < 3s для анализа
- **Пропускная Способность**: 1000+ новостей/минуту
- **Масштабируемость**: Горизонтальное и вертикальное масштабирование

### Надежность
- **Fail-Open Архитектура**: Продолжение работы при сбоях компонентов
- **Graceful Degradation**: Постепенное снижение функциональности
- **Отказоустойчивость**: Автоматическое восстановление после сбоев

### Качество Данных
- **Дедупликация**: Предотвращение повторной обработки
- **Валидация**: Проверка корректности данных на всех этапах
- **Свежеть**: Гарантированная актуальность данных

```bash

export CRYPTOPANIC_AUTH_TOKEN="your_token"

# 2. Запуск базовой конфигурации

# 4. Просмотр логов

docker logs news-analyzer --tail 20
```

### Базовое Использование в Сигналах

```python
from news_pipeline.enricher import NewsEnricher

# Создание enricher
enricher = NewsEnricher()

# Обогащение сигнала
def enrich_signal(signal_context):
    # Добавление новостного контекста
    enricher.attach(signal_context)

    # Проверка на критические новости
    if signal_context.news and signal_context.news.news_grade_id >= 4:
        return False  # Пропустить сигнал

    # Корректировка веса по риску
    if signal_context.news:
        risk_multiplier = 1.0 - (signal_context.news.news_risk * 0.5)
        signal_context.weight *= risk_multiplier

    return True
```

## Производственные Требования

### Ресурсы
- **CPU**: 2-4 cores для основных компонентов
- **RAM**: 4-8GB для LLM анализа, 2-4GB для остальных
- **Disk**: 50-100GB для логов и кеша
- **Network**: Стабильное интернет-соединение для API

### Зависимости
- **Redis 7.0+**: Для потоков и кеширования
- **Docker 24+**: Для контейнеризации
- **Python 3.11+**: Для ML компонентов
- **Go 1.22+**: Для высокопроизводительных компонентов

### Мониторинг
- **Prometheus**: Сбор метрик
- **Grafana**: Визуализация
- **ELK Stack**: Логирование и анализ
- **PagerDuty**: Алерты и инциденты
