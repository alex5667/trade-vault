# Глоссарий Scanner Infrastructure

Ниже приведены ключевые термины, сокращения и артефакты, используемые в проекте. Глоссарий помогает синхронизировать команды и ускоряет онбординг. При появлении новых доменных понятий обязательно пополняйте этот список.

---

## А

- **ADR (Architecture Decision Record)** — документ, фиксирующий архитектурное решение, варианты и мотивы. Хранятся в `docs/adr/`. Пример: `ADR-002-servemux.md`.
- **Alertmanager** — компонент Prometheus, отвечающий за маршрутизацию алертов в Slack, Telegram, PagerDuty.
- **ATR (Average True Range)** — индикатор волатильности, используется при выборе трейлингового профиля.
- **Auto Calibration** — автоматическая система оптимизации порогов сигналов на основе исторических данных.
- **Analytics V2/V3** — продвинутая система аналитики с ROC-кривым, threshold tuning и ML-интеграцией.

## Б

- **Binance WS** — WebSocket поток биржи Binance, основной источник тиков для криптоинструментов.
- **Book analytics** — сервис, анализирующий стакан (DOM) и вычисляющий показатели ликвидности.
- **Base OrderFlow Handler** — базовый класс для обработки Order Flow сигналов, реализующий spike detection и delta tracking.
- **Bootstrap Supervisor** [NEW] — сервис-страж, блокирующий работу экзекуторов до прохождения всех проверок здоровья (L3, UserStream, Projection).

## В

- **Vault** — система хранения секретов для продакшена. Экспортируется командой `make export-vault`.
- **Version (v: 1)** [NEW] — поле в событиях, гарантирующее паритет полей (qty/quantity) и совместимость контрактов.

## Г

- **Grafana** — система визуализации метрик и алертов. Дашборды находятся в `grafana_*.json`.
- **GPU Compute Service** — сервис ускорения вычислений с использованием CUDA и GPU для аналитики и калибровки.
- **G0–G15 Gates** [NEW] — многоуровневый пайплайн фильтрации сигналов (Data Health, Logic, ML, Risk).

## Д

- **DualRedis** — стратегия параллельной записи в два Redis-инстанса (primary + fallback) с автоматическим переключением при сбое.
- **Docker Compose профиль** — набор сервисов, описанный в дополнительном файле (`docker-compose.tp-trailing.yml`), позволяет быстро включать подсистемы.
- **Dataset Export** — система экспорта данных для ML моделей в форматах Parquet, CSV, HDF5.

## Е

- **Еhealtz/readyz** — HTTP эндпоинты сервисов, проверяющие жизнеспособность и готовность.

## Ж

- **Журнал MT5** — системный лог терминала MetaTrader 5, используется для диагностики исполнения команд.
- **Journal-First** [NEW] — архитектурный паттерн, при котором команды сначала пишутся в `orders:exec` журнал, а затем исполняются.

## З

- **Задержка (Latency)** — время прохождения события от одного этапа до другого. Мониторится через Prometheus метрики.

## И

- **Idempotency** — свойство операций давать одинаковый результат при повторных вызовах. В проекте обеспечивается ключами `sid`, `order_id`, `tp_event_id`.
- **Ingestion** — процесс приёма рыночных данных из внешних источников.

## К

- **Kline** — свечные данные из Binance, компонент тейльной обработки (не основной, но используется для аналитики).
- **Контрольный список (Checklist)** — формализованный набор шагов для релизов, операций и онбординга (см. `operations.md`).

## Л

- **Lag** — количество непрочитанных событий в Redis Stream относительно последнего подтверждённого.
- **LPUSH/RPOP** — паттерн работы с очередью в Redis: приоритетные элементы добавляются слева, читатель забирает справа.

## М

- **Makefile** — главный инструмент управления сервисами, тестами, линтерами и диагностикой.
- **MT5 Executor** — связка скриптов и HTTP адаптеров для выполнения команд на стороне MetaTrader 5.
- **ML Governance** [NEW] — система контроля качества моделей (дрейф PSI/KS, калибровка ECE/Brier).

## Н

- **Notify stream** — Redis Stream `notify:telegram`, через который Signal Performance Tracker отправляет сообщения в Telegram.
- **News Agent** [NEW] — LLM-агент для анализа новостей и извлечения `NewsTightenRecoDTO`.

## О

- **OBI (Order Book Imbalance)** — дисбаланс стакана ордеров, метрика ликвидности.
- **Observability** — практика наблюдаемости, включающая метрики, логи, алерты и дашборды.
- **Order queue** — Redis List `orders:queue`, содержащий команды на исполнение.
- **OrderFlow Handler** — специализированный обработчик для конкретного символа или группы символов.

## П

- **Postmortem** — документ, фиксирующий анализ инцидента, причину, последствия и улучшения.
- **Profiles:trailing** — набор конфигураций трейлинга, хранящийся в Redis Hash (`profiles:trailing:*`).
- **P4.1 Contract** [NEW] — унифицированный контракт задержек (t0-t5) с жёсткими SLO для Hot Path.
- **Projection Worker** [NEW] — воркер, материализующий состояние из журнала `orders:exec` в БД или кеш.

## Р

- **Regime** — классификация состояния рынка (momentum/ranging/high volatility). Вычисляется `regime-worker`.
- **Roadmap** — план развития системы, см. `roadmap.md`.
- **ROC (Receiver Operating Characteristic)** — кривая для оценки качества бинарной классификации сигналов.

## С

- **SID (Signal ID)** — уникальный идентификатор сигнала: `signal-<symbol>-<timestamp>-<source>`.
- **Signal Performance Tracker** — сервис аналитики сигналов, формирующий отчёты и уведомления.
- **ServeMux** — роутер Go 1.22, используемый в `go-gateway`.
- **SLA/SLO** — метрики качества обслуживания; SLA — обязательный уровень, SLO — целевой показатель.

- **TP1/TP2** — уровни Take Profit. TP1 активирует трейлинг.
- **Threshold Tuning** — автоматическая оптимизация порогов для генерации сигналов.
- **Tick** — рыночное событие (цена, объём). Источники: Binance WS, MT5.
- **t0–t5** [NEW] — критические временные точки P4.1 (Ingest, Redis Write, Logic Start/End, Journal Ack, Execution).
- **Trailing profile** — параметры смещения стоп-лосса после TP1.

## У

- **Уведомления (Notifications)** — сообщения в Telegram/Slack, формируемые tracker/Alertmanager.
- **Weak Progress** — паттерн медленного движения цены в направлении сигнала, требующий специальной обработки.

## Ф

- **FastAPI** — Python фреймворк для HTTP сервисов (`tick_ingest_server`, `book_analytics_service`).

## Х

- **Healthcheck** — проверка состояния сервиса (`/healthz`, `/readyz`, `make diagnose`).

## Ч

- **Чек-лист релиза** — набор шагов, выполняемый перед деплоем/после (`operations.md#управление-релизами`).

## Ш

- **Шардирование** — распределение нагрузки по нескольким инстансам/символам (используется в Signal Hub, Trailing).

## Э

- **Экспортер** — компонент, выдающий метрики Prometheus (например, встроенный в Go/Python сервисы).

## Ю

- **Юнит-тесты** — тесты модулей. Запускаются `make test-go`, `make test-python`.

## Я

- **Якорь (Anchor)** — фиксированная точка в последовательности событий, например, `TP1_HIT` выступает якорем для трейлинга.
