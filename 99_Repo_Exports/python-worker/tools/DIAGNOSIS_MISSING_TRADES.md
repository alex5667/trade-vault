# Диагностика: Почему не открываются сделки для 1000PEPEUSDT, 1000SHIBUSDT, 1000FLOKIUSDT, 1000BONKUSDT

## Проблема

Для символов `1000PEPEUSDT`, `1000SHIBUSDT`, `1000FLOKIUSDT`, `1000BONKUSDT` выводится сообщение:
```
- недостаточно данных для рекомендаций (found_trades=0-4, filtered_wins_all=0-1, filtered_wins_trailing=0-1, need_trades=40, need_wins~=15).
```

## Анализ кода

### Как работает система рекомендаций

1. **Загрузка сделок** (`load_trades_from_stream`):
   - Читает из Redis stream `trades:closed`
   - Фильтрует по `source` (по умолчанию `CryptoOrderFlow`) и `symbol`
   - Сканирует до 2M записей для поиска достаточного количества сделок

2. **Фильтрация** (`_count_filtered_wins`):
   - Фильтрует сделки по:
     - `symbol` и `source` должны совпадать
     - `one_r_money > EPS` (должен быть рассчитан риск)
     - `winners_only=True`: только выигрышные сделки (`pnl_net > 0`)
     - `mfe_pnl > 0` (должен быть максимальный благоприятный экскурс)
     - `mfe_r <= 100.0` (защита от выбросов)

3. **Проверка достаточности данных** (`recommend_trailing_size`):
   - Требуется: `count_total >= min_trades` (по умолчанию 40)
   - И: `wins >= effective_min_wins` (по умолчанию ~15)

### Возможные причины проблемы

#### 1. **Нет закрытых сделок в stream**
   - Сигналы не генерируются для этих символов
   - Сигналы генерируются, но ордера не открываются
   - Ордера открываются, но сделки не закрываются (застряли в `orders:open`)

#### 2. **Несоответствие source**
   - Сделки в stream имеют другой `source` (например, `cryptoorderflow` вместо `CryptoOrderFlow`)
   - Код нормализует source через `canon_source()`, но возможны варианты

#### 3. **Недостаточно данных**
   - Есть сделки, но их меньше 40
   - Есть сделки, но недостаточно выигрышных (< 15)

## Решение

### Улучшения в коде

1. **Расширенная диагностика** (уже добавлено):
   - Показывает все найденные `source` для символа
   - Предлагает проверить другие source
   - Даёт рекомендации по проверке сигналов и открытых позиций

2. **Диагностический скрипт** (`diagnose_missing_trades.py`):
   - Проверяет `trades:closed` stream
   - Проверяет `orders:open` (открытые позиции)
   - Проверяет сигналы в `signals:*` streams
   - Проверяет статистику в `stats:*`

### Что проверить

1. **Проверьте сигналы**:
   ```bash
   # В Redis проверьте streams сигналов
   redis-cli XREVRANGE signals:cryptoorderflow + - COUNT 1000 | grep -i "1000PEPE\|1000SHIB\|1000FLOKI\|1000BONK"
   ```

2. **Проверьте открытые позиции**:
   ```bash
   # Проверьте, есть ли застрявшие позиции
   redis-cli SMEMBERS orders:open
   # Для каждой позиции проверьте детали
   redis-cli HGETALL order:<order_id>
   ```

3. **Проверьте все source в stream**:
   ```bash
   # Запустите с расширенной диагностикой
   python3 tools/recommend_trailing_from_redis.py \
     --symbols 1000PEPEUSDT,1000SHIBUSDT,1000FLOKIUSDT,1000BONKUSDT \
     --source CryptoOrderFlow
   ```

4. **Попробуйте другие source**:
   ```bash
   # Если найдены сделки с другим source
   python3 tools/recommend_trailing_from_redis.py \
     --symbols 1000PEPEUSDT \
     --source <найденный_source>
   ```

5. **Проверьте логи**:
   - Логи генерации сигналов
   - Логи открытия ордеров
   - Логи закрытия сделок (TradeMonitor)

### Возможные исправления

1. **Если сигналы не генерируются**:
   - Проверьте конфигурацию символов в orderflow handler
   - Проверьте фильтры по символам
   - Проверьте, активны ли эти символы в системе

2. **Если ордера не открываются**:
   - Проверьте `orders:queue` - попадают ли туда сигналы
   - Проверьте логи executor'а
   - Проверьте фильтры качества исполнения

3. **Если сделки не закрываются**:
   - Проверьте `orders:open` - есть ли застрявшие позиции
   - Проверьте логи TradeMonitor - обрабатываются ли события закрытия
   - Проверьте таймауты orphan позиций

4. **Если source не совпадает**:
   - Используйте правильный source при запуске
   - Или исправьте source при сохранении сделок в stream

## Следующие шаги

1. Запустите диагностический скрипт (если Redis доступен)
2. Проверьте логи системы на наличие ошибок
3. Проверьте конфигурацию символов
4. Используйте улучшенную диагностику в `recommend_trailing_from_redis.py`
